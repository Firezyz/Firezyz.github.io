复杂系统是如何失效的

发表于 2017-06-02   |   分类于 技术   |  

原文地址

http://web.mit.edu/2.75/resources/random/How%20Complex%20Systems%20Fail.pdf

复杂系统本质上是有危险的系统

所有引发人们关注的系统(例如交通运输系统,医疗保健系统,发电系统)就其本身的属性而言都有内在且不可避免的危险.危险暴露的频率有时候会发生变化,但系统中所涉及的过程本身就是固有且不可约减的危害.这些危害的存在驱动着我们对这些系统的危害进行防护.

复杂系统能够成功抵抗失效

故障的严重后果推进了我们建立多层的故障规避措施.这些防御措施包括熟知的技术部分(例如备份系统,设备的“安全”特征)和人员部分(例如培训,知识), 还包括一些整理成文的规避措施(例如规章制度,认证,工作守则,团队训练).这些措施的效果是提供了一系列规避事故的保护.

灾难需要多次故障 - 单点故障是不够的

因为多重保护是生效的,所以系统操作通常是成功的.明显的灾难性故障在起初发生时候通常是比较小的,但是很明显的无害小问题会造成系统性意外情况的出现.这些小故障都是导致灾难的必要条件,但逐步累积到一定量级才足以产生灾难性故障.换句话说,除了明显暴露出来的系统事故,其实还存在更多的失败的可能.大部分的初期失效都会被预先设计的系统安全组件封锁.操作层面的问题几乎都会被系统的维护人员规避掉.

复杂系统包含了不断变化的潜在故障混合物

系统的复杂性使得它们即使有多个缺陷,也可以正常运行.因为这些问题单独存在不足以造成故障,以至于在运行中被认为是故障的次要因素.之所以没有事前消除所有的潜在故障,主要受到经济成本的限制,以及事前很难猜到这种故障是怎么造成最后的事故的.同时,由于技术更替,工作组织的变化以及为了消除故障所做的一些工作,也会导致故障的原因不断在变化.

复杂系统有降级模式

前一点的推论是,复杂系统可能已经是支离破碎的.系统尽管存在很多缺陷,但是因为系统存在各种各样的维护的补丁,所以系统仍然可以继续运作.故障复盘之后,几乎总能发现该系统有一个“原始事故”的历史.在事故发生之前,这些不断恶化的问题应该已经被争论好多次了.系统操作是组件(组织,人力,技术)出问题并被不断更换的一种动态平衡.

灾难总是触手可及

复杂系统具有灾难性故障的隐患.人类从业人员几乎总是紧密接触这些潜在的故障 - 灾难可以在任何时间和几乎任何地方发生.灾难性后果的隐患是复杂系统的标志.消除这种灾难性失败的可能性是不可能的;这种失败的隐患始终以系统本身的性质存在.

事故后,发生事故的根本原因是“基础性”错误

由于明显的失效是有多重失效累积而成,因此并不存在孤立的造成事故的原因.多个因素都对事故的发生造成影响.每一个”先天性不足”本身就可以造成事故的发生.只有多个失效累积在一起才足以造成事故.的确,造成事故的原因之间是有关联的,多种不同的情况叠加在一起会引发事故.因此不存在孤立的造成事故出现的根本原因是完全有可能的.在这些推论的基础上,我们对根本原因的评估并不反映出失效属性的技术理解,这些评估仅仅反映的是造成这些事故的社会原因和文化原因.

事后事故分析是有偏差的

在任何时间对从业人员而言,了解事故后果实际上比了解事故的原因更为重要.这意味着人为表现的事后回溯事故分析是错误的.对事故的认识能够使事故后观察人员在再次发生因为同样因素而造成的事故之前刷新从业人员对事故的见解.从业人员似乎“本应当知道”引发事故不可避免的因素.“事后诸葛亮”是对事故开展调查的主要障碍,尤其是在涉及专家人为表现时.

操作员的双重角色:失效的生产者和失效的防御者

系统从业人员操作系统制造其所期望的产品,这些人员同样需要预先阻止事故的发生.系统的操作是动态的,生产与初期失效可能性之间的平衡不可避免.
局外人很少能够了解该角色的双重性.在非事故时间期间强调的是作为生产者的角色.在发生事故之后则强调的是防御失效的角色.不管在哪一时间段,局外人的观点会误解操作员在这两种角色之间稳定的同时参与.

所有从业人员的行为都是赌博

在发生事故之后,明显的故障看起来通常是在所难免的,错误的参与者的行为或者参与者故意的行为会使得即将发生的某些故障被忽视.但是所有参与者的行为实际上都是在赌博,也就是在面对不确定结果时候所采取的一些行动.不确定性的程度一直是在不断变化的.参与者的行为就是赌博,并且在事故发生之后表面的更为明显.一般而言,事后分析会将这些赌博性行为视为是非常糟糕的行为.但是与之相反的是,那些成功的结果通常也是赌博性行为的结果,无法被人们广泛领会.

处于前线位置的行为能够解决所有的歧义

组织是含糊不清的,在生产目标、资源的有效使用、经济和运营成本以及将高重要性和低重要性事故的可接受风险联系之间通常也是有意为之的.从业人员在系统的前线位置所采取的行为能够解决所有的歧义性。在事故发生之后，从业人员的行为可以视为是“错误”或“违规行为”，但是这些评估带有很大的“事后诸葛亮”的倾向，且忽视了其他的驱动性因素，尤其是生产压力。

从业人员是复杂系统的一个适应性较强的因素

从业人员和一线管理人员都会积极适应系统,以实现生产最大化,并将事故的发生降低到最低程度.这种适应通常发生在每时每刻.部分此类适应包括:

1. 对系统进行调整以降低易损部件发生故障的可能性;
2. 在预期的高需求领域精选关键资源;
3. 提供从预期内和预期外故障进行故障恢复的方法;
4. 建立早期检测系统性能变化的方式,允许合理的削减生产或以其他形式增强系统的弹性.

复杂系统中从业人员的专业知识一直是不断变化的

在复杂系统的运营和管理过程中需要大量的专业知识.但是由于需要不时更换专业人员,这些专业知识会随着技术的变革发生相应的变化.在任何一种情况下,技能和专业知识的培训和强化本身就是系统功能的一个组成部分.因此,在任何情况下,给定的复杂系统会含有从业人员和实习生所需要的不同层面的专业知识.与专业知识有关的关键问题主要来自于如下两个方面:

1. 对于异常困难或要求苛刻的产品需求而言利用有限的专业知识作为资源;
2. 专业知识拓展需求以备未来不时之需.

系统升级会引入新形式的故障

可靠系统中明显事故降低的发生率可以鼓励变革,尤其是鼓励新技术的使用,以降低低后果和高频失效出现发生频率的重要性.这些变革实际上为新的低频但较高的结果失效提供了机遇.当使用新技术消除人们对系统理解的误区并实现高精度系统性能时,新技术通常会引入新形式的大规模灾难性故障.这种情况并不罕见,这种新的但发生频次较低的灾难会对新技术的淘汰产生更为重要的影响.这些新形式的失效在发生之前几乎无法看到,因此在推定变革有益的特征时需十分小心.由于这些新的且会造成严重后果的事故发生的频率非常低,因此我们可以在事故发生之前对多个系统进行变更,使因技术的引入而产生的失效很难发生.

限制预防未来事件时效性原因的角度

“人为失误”的事故后补救措施通常以妨害“引发”事故的活动为基础.这些终端链措施几乎不会降低未来发生事故的可能性.事实上,完全相同的事故的可能性出现的概率是非常低的,因为失效的潜在模式是一直在变化的.除了系统的安全性不断提高之外,事故后补救措施通常也能加强系统的耦合性和复杂度.这种潜在事项数量的增加也是的检测并封锁事故轨迹变得更为困难.

安全性是系统的特征,而非系统组件的特征

安全性是系统的突出特征;安全性不会存在于某一个人,设备装置,组织或系统的某一部分中.安全性无法购买,无法加工制造.安全性并不是独立于系统其他组件的一个特征.这意味着,安全性无法像原材料或原料进行操作.在任何系统中,安全性的状态永远是动态的;连续的系统变化确保了危害以及危害的管理也在不断的变化当中.

人们不断发展安全性

与系统操作无任何关联的失效是由操作人员在可容忍性能边界之内操作系统实现的.在很大程度上,这些活动是能正常操作的一个组成部分.但是由于系统操作不可能是无故障的,因此人从业人员必须能够适应这种不断变化的状态,并不断强化和完善系统的安全性.这种适应通常相当于从大量的可用资源中选择经过反复预演的资源,但是,有时候这种适应是新方法重新创造的过程或是现有多种方法的组合.

无故障运行需要故障处理经验

认识到危害并正确方式运行系统来维持系统可接受的性能需要与失效或故障亲密接触.更为鲁棒的系统性能很有可能是操作人员能够识别并理解失效原因的系统.这是情况系统性能开始退化的开始,这会使预测变得更为困难,或者使操作人员无法简单的从故障中恢复.在本质上是有危害的系统中,操作人员会遇到一些影响到系统总体性能的危害.系统改良过的安全性取决于向操作人员提供经过校准的危害的看法.这同样取决于操作人员的动作能否使系统回到正常的运行状态中.
